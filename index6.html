2. Algorithms & Data Structures

ðŸ”¹ Convolutional Neural Networks (CNNs)

Convolutional Neural Networks (CNNs) are a type of deep learning model particularly effective for image-related tasks. In the context of GE Healthcare, CNNs are used to process and analyze medical scans such as X-rays and MRIs. CNNs work by applying convolutional filters that slide over the input image to capture features like edges, textures, and patterns [3].

CNNs enhance the system by eliminating the need for manual feature extraction. Unlike traditional ML models that require hand-crafted features, CNNs learn what to extract directly from the raw data [3].

ðŸ”¹ U-Net for Image Segmentation

U-Net is a specialized CNN architecture designed for biomedical image segmentation. Unlike classification models that predict a label for an entire image, segmentation models assign a label to each pixel [2].

U-Net follows an encoder-decoder structure: the encoder compresses the image to learn context, while the decoder reconstructs it to match the original resolution, producing a segmented output. Skip connections between encoder and decoder layers preserve spatial details [2].

ðŸ”¹ Sagemaker Pipelines (DAG-based)

AWS SageMaker Pipelines enable building and automating ML workflows using Directed Acyclic Graphs (DAGs). Each stepâ€”data preprocessing, model training, evaluation, and deploymentâ€”is a node in the graph [4].

In GE Healthcare's workflow, raw images are stored in Amazon S3 and passed through a labeling stage using Ground Truth. The labeled data is preprocessed and sent to SageMaker for training CNN or U-Net models [4].
